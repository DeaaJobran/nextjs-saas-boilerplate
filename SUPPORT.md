# Support

## Where To Ask

- Use GitHub Issues for confirmed bugs, documentation gaps, and accepted feature requests.
- Use GitHub Discussions for questions, proposals, and implementation tradeoffs.
- Use private vulnerability reporting for security issues.

## What To Include

For bugs:

- Reproduction steps.
- Expected behavior.
- Actual behavior.
- Environment details.
- Logs, screenshots, or failing tests when useful.

For feature requests:

- Problem statement.
- Proposed API or user workflow.
- Why this belongs in the boilerplate instead of a downstream app.
- Security, maintenance, and documentation implications.

## Scope

Maintainers prioritize issues that improve the reusable boilerplate foundation. Product-specific customizations may be redirected to discussions or closed if they do not fit the project scope.
