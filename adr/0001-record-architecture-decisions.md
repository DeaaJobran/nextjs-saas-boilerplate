# ADR 0001: Record Architecture Decisions

- Status: Accepted
- Date: 2026-07-06

## Context

The project is a greenfield, open-source SaaS boilerplate with a large feature surface. Architecture choices need to remain visible so future contributors understand why foundational decisions were made.

## Decision

Architecture decisions will be recorded as ADRs under `adr/`. Each ADR should include status, date, context, decision, consequences, and references when relevant.

## Consequences

- Major stack and architecture changes require a tracked decision.
- The project can evolve without relying on private planning notes.
- Superseded decisions should be replaced by new ADRs instead of edited silently.
